Slide 4

Routing

React Router DOM v7 · Public routes redirect authenticated users · Protected routes redirect guests

PUBLIC /  /signup Signup page — redirects to /employees if already signed in
PUBLIC /signin Signin page — redirects to /employees if already signed in
PROTECTED /employees Employee management table
PROTECTED /products Product catalog table
PROTECTED /tasks Task board (Kanban-style)

ProtectedRoute

Uses Clerk useUser() — redirects to /signin if not signed in

PublicRoute

Redirects authenticated users away from auth pages to /employees

State Management

🗄️ Server State — TanStack Query

  • Used in Products feature
  • useQuery with keepPreviousData for smooth pagination
  • useMutation + automatic cache invalidation
  • DevTools included in development

🔧 Local State — useState/useEffect

  • Used in Employees + Tasks
  • Manual fetch in useEffect
  • refreshTrigger counter for re-fetching
  • Opportunity: migrate to React Query

Custom Hooks

Shared / Utility

useIsAdmin()

Checks user.publicMetadata.role === "admin" via Clerk — controls write access in UI

useDebounce(value, delay)

Generic 1000ms debounce for search inputs across all pages

useOutsideClick(ref, cb)

Dismisses dropdowns & menus when clicking outside

useTheme() · useSidebar()

Context consumers for theme and sidebar state

Feature Hooks

useEmployees()

All employee page state — data, pagination, search, modal type, CRUD handlers

useProducts()

Same scope but backed by React Query — cleaner cache invalidation

useTask()

Fetches all tasks on mount; exposes tasks, setTasks, isLoading

useSigninForm() · useSignupForm()

Form state + Clerk submission logic encapsulated per auth flow

Slide 11

Authentication Flow

🧩 ClerkProvider — wraps entire app at root
↓
👤 User visits /signup or /signin → Clerk handles session creation
↓
✅ RouteGuard reads useUser().isSignedIn → gates protected routes
↓
🔑 useIsAdmin() checks user.publicMetadata.role === "admin" → shows/hides write actions

🚫 Non-Admin Users

  • Can view all pages
  • Add / Edit / Delete buttons hidden
  • Read-only access to all data

✅ Admin Users

  • Full CRUD on Employees & Products
  • Role set in Clerk's publicMetadata
  • No backend role config needed client-side

const { data, isLoading } = useQuery({
  queryKey: ["products", pageIndex, pageSize, debouncedSearch],
  queryFn: () => getProducts({ limit, offset, search }),
  placeholderData: keepPreviousData,   // no flash on page change
});

const deleteMutation = useMutation({
  mutationFn: deleteProduct,
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ["products"] });
    onSuccess("Product deleted successfully");
  },
});

Slide 13

Key Observations

✅ Strengths

  • Vertical slice architecture — each feature is self-contained
  • Shared generic components (Table, Modal, Pagination)
  • URL-driven table state — bookmarkable & shareable
  • Clerk handles all auth complexity
  • Zod + react-hook-form = type-safe forms
  • Code quality tools: ESLint, Prettier, Husky

⚠️ Opportunities

  • Employees + Tasks still use useState/useEffect — migrate to React Query
  • No global loading/error boundary
  • Base URL hardcoded — should use env var
  • Notification list is static (no real-time)
  • No unit or integration tests present
